# 产线规划功能方案

> 目的：把现有两阶段 MILP 求产线规模的结果，进一步生成一个整体物流图。  
> 方法：对每个物品单独构建运输/分配子问题（供给点、需求点按机器粒度），在满足所有需求的前提下最小化连接数量，再把所有物品的结果合并成一张图。

## 0. 范围与非目标

### 0.1 目标
- 输入：`Catalog` + `AicInputs` + 求解结果。
- 输出：
  - 每个物品的一组物流连接（供给点 → 需求点，带流量）。
  - 合并后的整体物流图。

### 0.2 不做
- 不优化空间距离/布局长度，只优化边数量。
- 不做时序、缓冲、库存动态，只做稳态流量（units/min）。
- 不引入第二种求解策略，先只实现 MILP。

## 1. 关键定义

### 1.1 连接
- 供给点 `s` → 需求点 `d` 传输物品的一条边。
- 计数规则：如果某条边上该物品的流量 `f_{s,d} > 0`，则认为该边被使用，计 `1` 条连接。

### 1.2 节点
- 内部求解：按**每台机器实例**做供给/需求点。
- 输出展示：仍按同配方机器组聚合，但必须保留未满速的区分：
  - `Full`：满速机器
  - `Partial`：未满速机器（需要单独显示）
